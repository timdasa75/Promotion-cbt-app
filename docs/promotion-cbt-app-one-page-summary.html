Promotion CBT App - One-Page Summary

Repo-based summary generated from README, index.html, js modules, data/topics.json, and functions/index.js.

What It Is

Promotion CBT is a browser-based practice platform for Nigerian Federal Civil Service promotion preparation (directorate track). It combines topic drills, timed exam simulation, and post-quiz performance feedback in a static web app with optional Firebase-backed cloud auth.

Who It's For

Primary persona: Federal Civil Service officers preparing for directorate-level promotion CBT who need structured, repeatable study across policy, regulation, leadership, ICT, and reasoning topics.

What It Does

  • Loads a 10-topic question bank from data/topics.json and topic JSON files.
  • Supports Practice, Exam, and Review modes, with per-mode behavior and feedback.
  • Runs a 40-question cross-topic Directorate Mock Exam (premium-gated).
  • Captures retry queues for missed questions and allows focused re-attempts.
  • Tracks attempts, streak, weakest-topic recommendation, and results analytics.
  • Provides keyboard quiz controls (A-D/1-4, arrows, Enter) for faster navigation.
  • Includes admin tooling for plan overrides, upgrade request review, and user directory/status operations.

How It Works (Architecture)

  • UI shell: index.html + CSS + ES modules; js/boot.js sets base path and runtime auth requirements.
  • App orchestration: js/app.js initializes data, manages screens, binds auth/admin/dashboard actions.
  • Data pipeline: js/data.js loads topics; js/topicSources.js fetches topic files and normalizes subcategory/question shapes.
  • Quiz engine: js/quiz.js builds sessions, timers, scoring, review filters, mock exam composition, and persistence.
  • Auth/entitlements: js/auth.js handles local users/session storage plus Firebase REST + Firestore profile sync when configured.
  • Admin backend: functions/index.js exposes HTTPS admin list/delete endpoints and auth-profile delete sync triggers.
Data flow: Browser load -> boot/runtime config -> topics fetch -> login (local/cloud) -> topic/category selection -> question extraction -> quiz scoring/persistence -> dashboard + optional cloud/admin sync.

How To Run (Minimal)

  1. Install dependencies: npm install
  2. Serve as static site: python -m http.server 4173
  3. Open: http://127.0.0.1:4173/
  4. Optional for cloud auth: configure config/runtime-auth.js using config/runtime-auth.example.js.
Not found in repo: a dedicated npm start/npm run dev script for local serving.
